# ローカル環境でのUIプレビューガイド

## 📋 概要

ローカル環境では、各システム（document-automation、amazon-analyticsなど）は起動していないため、実際の画面遷移は確認できません。

しかし、各システムのテンプレートファイル（HTML）を直接ブラウザで開くことで、ヘッダーのスタイルやデザインを確認できます。

---

## 🎯 確認できる項目

- ✅ ヘッダーのデザイン（統一スタイル）
- ✅ 戻るボタンの表示
- ✅ バージョン情報の表示
- ✅ レイアウトの確認

---

## 🚀 方法1: テンプレートファイルを直接ブラウザで開く

### 手順

#### 1. 各システムのテンプレートファイルを確認

```bash
# テンプレートファイルの場所を確認
find /Users/Yoshi/nas-project -name "index.html" -path "*/templates/*"
```

#### 2. テンプレートファイルをブラウザで開く

**注意**: テンプレートファイルにはJinja2の変数（`{{ variable }}`）が含まれているため、そのまま開くと正しく表示されません。

**解決策**: テンプレートファイルを一時的にHTMLファイルとして保存し、変数を実際の値に置き換えます。

## 🎯 推奨方法: NAS環境で確認

ローカル環境では完全な動作確認は難しいため、**NAS環境で実際の画面遷移を確認することを推奨**します。

### NAS環境での確認手順

1. **各システムの変更をコミット・プッシュ**
   ```bash
   cd /Users/Yoshi/nas-project
   git add .
   git commit -m "fix: ヘッダーを統一スタイルに変更、同じタブで開くように修正"
   git push origin feature/monitoring-fail2ban-integration
   ```

2. **NAS環境でデプロイ**
   ```bash
   # NASにSSH接続
   ssh -p 23456 AdminUser@192.168.68.110

   # 最新コードを取得
   cd ~/nas-project
   git pull origin feature/monitoring-fail2ban-integration

   # 各システムを再ビルド
   cd nas-dashboard
   docker compose restart

   cd ../document-automation
   docker compose restart

   # 他のシステムも同様に再起動
   ```

3. **ブラウザで確認**
   - ダッシュボード: https://yoshi-nas-sys.duckdns.org:8443/
   - 各システムの画面遷移を確認

- [ ] ヘッダーのスタイルが統一されているか

## ⚠️ 注意事項

1. **ローカル環境の制限**
   - 各システムは起動していないため、実際の画面遷移は確認できません
   - 認証機能も動作しません
   - APIエンドポイントは利用できません

2. **NAS環境での確認が推奨**
   - 完全な動作確認はNAS環境で行う必要があります
   - 実際の使用環境での確認が最も重要です

3. **テンプレートファイルの変数**
   - テンプレートファイルにはJinja2の変数が含まれているため、そのままブラウザで開いても正しく表示されません
   - 変数を実際の値に置き換えたHTMLファイルを作成する必要があります
