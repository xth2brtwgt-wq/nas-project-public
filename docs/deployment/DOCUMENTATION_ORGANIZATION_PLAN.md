# ドキュメント整理計画

## 📊 現状分析

- **総MDファイル数**: 約244個（`docs/deployment`のみ）
- **主な問題点**:
  - トラブルシューティングドキュメントが多数（完了済みの問題の記録）
  - 重複する内容のドキュメント
  - 完了報告が散在
  - カテゴリ別の整理が不十分

## 🎯 整理方針

### 1. ディレクトリ構造の再編成

```
docs/deployment/
├── README.md                    # インデックス
├── guides/                      # ガイド・手順書
│   ├── setup/                   # セットアップガイド
│   ├── configuration/           # 設定ガイド
│   └── troubleshooting/         # トラブルシューティング（現在進行形）
├── reports/                     # 完了報告・サマリー
│   ├── completed/               # 完了済みタスク
│   └── summaries/               # サマリードキュメント
├── archive/                     # アーカイブ（古いトラブルシューティング）
│   ├── auth/                    # 認証関連の古い記録
│   ├── nginx/                   # Nginx関連の古い記録
│   └── deployment/              # デプロイメント関連の古い記録
└── templates/                  # テンプレート
```

### 2. カテゴリ別分類

#### ガイド・手順書（guides/）
- セットアップガイド
- 設定ガイド
- 現在進行形のトラブルシューティング

#### 完了報告（reports/）
- 完了済みタスクの記録
- サマリードキュメント

#### アーカイブ（archive/）
- 古いトラブルシューティング記録
- 完了済みの問題の詳細記録

### 3. 統合対象

#### 認証関連（多数の重複）
- `DASHBOARD_AUTH_*` (約30個) → 統合して1-2個に
- `MEETING_MINUTES_AUTH_*` (約15個) → 統合して1-2個に
- `YOUTUBE_TO_NOTION_AUTH_*` (約10個) → 統合して1-2個に
- `DOCUMENT_AUTOMATION_MONITORING_AUTH_*` (約6個) → 統合して1個に

#### Nginx関連（多数の重複）
- `NGINX_PROXY_MANAGER_STATIC_FILES_404_*` (約10個) → 統合して1-2個に
- `NGINX_PROXY_MANAGER_*` (約30個) → 主要なもののみ残す

#### 完了報告（散在）
- `*_COMPLETE.md`, `*_SUCCESS.md`, `*_SUMMARY.md` → `reports/`に移動

### 4. 削除候補

- 古いトラブルシューティング記録（完了済み）
- 重複する内容のドキュメント
- 一時的なデバッグ記録

## 📋 整理手順

### Phase 1: ディレクトリ作成
1. 新しいディレクトリ構造を作成
2. インデックスファイルを作成

### Phase 2: 分類・移動
1. カテゴリ別に分類
2. 適切なディレクトリに移動
3. 重複ドキュメントを統合

### Phase 3: アーカイブ
1. 古いトラブルシューティング記録をアーカイブ
2. 完了報告を統合

### Phase 4: インデックス更新
1. README.mdを更新
2. 各カテゴリのインデックスを作成

## ✅ 期待される結果

- **整理後**: 約50-80個のMDファイル（アクティブなもの）
- **アーカイブ**: 約150-200個（参照用に保持）
- **可読性**: 大幅に向上
- **保守性**: 大幅に向上
