# プロジェクト概要と基本原則

## プロジェクト情報
- このプロジェクトはNAS(UGreen DXP2800)環境で稼働するツールのプロジェクトです

## 基本原則
- 常に可読性を重視し、自己説明的なコードを書く
- SOLID原則に従った設計を心がける
- DRY（Don't Repeat Yourself）原則を守る
- YAGNI（You Aren't Gonna Need It）原則を適用し、必要以上の機能は追加しない
- シンプルで保守しやすいソリューションを優先する

## 禁止事項
- any型の使用禁止（unknownを使う）
- console.log/console.errorを本番コードに残さない
- アロー関数でのreturn省略（可読性のため明示的に書く）
- マジックナンバー（数値は定数化する）
- 深いネスト（3階層以上は関数に分割）

## 生成時の注意
- 既存のコードスタイルに合わせる
- 変更理由をコメントで説明する
- パフォーマンスへの影響を考慮する
